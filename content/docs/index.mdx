## Xena Studios

Xena Studios builds **open-source infrastructure for Minecraft networks and self-hosted
game hosting**. Our plugins are designed to run on every server in a network with near-zero
overhead, fail safe on bad input, and stay thread-correct on both Paper and Folia. Raptor
extends that philosophy off the game server and onto the boxes it runs on.

Everything we publish is open source on [GitHub](https://github.com/xena-studios), and our
Paper/Folia plugins are available on [Modrinth](https://modrinth.com/organization/xena-studios).

## Projects

### Plugins

- [**xUtilities**](/docs/xutilities) — a lightweight, high-performance utility-command plugin
  for Paper (and Folia). The small quality-of-life commands almost every server wants, each
  individually toggleable, permissioned, and fully customizable.
- [**xLimbo**](/docs/xlimbo) — a rock-solid fallback / limbo server plugin for Paper networks.
  When the main server goes down, players land in a safe void world instead of being
  disconnected.
- [**NeoSkript**](/docs/neoskript) — a fast, modern, drop-in replacement for Skript, built for
  Paper + Folia on Java 25 with a clean, extensible API.

### Platform

- [**Raptor**](https://www.raptorpanel.net) — a hosted, multi-tenant control panel for running
  Docker game servers and apps on your own Linux boxes, plus the per-box daemon that does the
  work. Learn more at [raptorpanel.net](https://www.raptorpanel.net).
